更多启动配置

MLL提供了各种启动Minecraft的选项。本页显示了最重要的内容。有关完整列表，请查看模块-命令的文档。

JVM参数

jvmArguments是一个字符串列表。每个参数都是列表中的一个条目。以下是一个例子：

# 正确使用方法
options["jvmArguments"] = ["-Xmx2G", "-Xms2G"]

# 错误案例1
options["jvmArguments"] = ["-Xmx2G -Xms2G"]

# 错误案例2
options["jvmArguments"] = "-Xmx2G -Xms2G"

确保每个参数都以”-“开头，否则Java会出现“找不到或加载主类”错误。

Java路径

这个选项设置了启动Minecraft时用的Java路径。如果游戏版本的json里包含了Java的下载链接，它将下载并使用这个版本。否则它将只单纯的使用java命令。MLL允许覆盖此内容。如果您想启动一个需要旧版本Java且json文件中不包含旧版本Java的下载链接，这可能会很有用。

options["executablePath"] = "Java/的/路径"

自定义分辨率

MLL允许使用自定义分辨率启动Minecraft。您首先需要启用自定义分辨率。然后，您就可以设置它了：

# 启用自定义分辨率
options["customResolution"] = True
# 设置自定义分辨率
# 宽度
options["resolutionWidth"] = "600"
# 高度
options["resolutionHeight"] = "500"

请确保您使用的字符串而不是数字。

游戏目录

游戏目录是Minecraft保存其游戏内容（如世界、资源包、选项）的目录。默认情况下，您的.minecraft目录被用作游戏目录。

如果你要做版本隔离，这对你很有帮助。

options["gameDirectory"] = "你/的/游/戏/路/径"

如果这个目录不存在，Minecraft会自动创建它。

使用演示模式

Minecraft有一个内置的演示模式，相当于官方启动器的试用版本。

MLL允许您启用演示模式。您至少需要Minecraft版本1.3.1才能使用演示模式。

options["demo"] = True
