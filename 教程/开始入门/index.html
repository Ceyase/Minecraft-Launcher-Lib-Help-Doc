开始入门

文档的第一篇将介绍如何安装和运行Minecraft。这里跳过了微软登录。

Minecraft根目录

想要开始使用MLL库，我们首先需要一个.minecraft目录。我们可以新建一个Minecraft目录或者使用默认目录。您可以使用minecraft_launcher_lib.utils.get_minecraft_directory()获取默认目录。

# 获取您系统的默认Minecraft目录
minecraft_directory = minecraft_launcher_lib.utils.get_minecraft_directory()
# 或者你也可以使用你自己的目录
# minecraft_directory = "路径/到/你的/.minecraft/文件夹"
# 比如
minecraft_directory = "D:/PCL/.minecraft"

安装Minecraft

在你启动Minecraft之前，你需要先安装它. 这可以使用install_minecraft_version命令完成. 这里假设我们想在Minecraft目录中安装1.17。

minecraft_launcher_lib.install.install_minecraft_version("1.17", minecraft_directory)

为了知道如何安装Minecraft, MLL会首先在Minecraft目录中查找你Minecraft版本的JSON文件. 在1.17的情况下， 它通常是minecraft_directory/versions/1.17/1.17.json。通过这种方式安装能安装魔改版本。（例如Impact）如果JSON文件不存在，MLL会尝试从Mojang的服务器下载它. install_minecraft_version 确保了Minecraft版本的安装正确，因此您需要在每次启动Minecraft之前都调用一次它，即使您已经安装了该版本。

获取Minecraft版本

你总不能让玩家每次只能游玩到一个版本吧。 minecraft_launcher_lib.utils.get_available_versions(minecraft_directory)能返回Minecraft文件夹中所有能用的版本. 它通常返回这样的一个列表:

[
    {
        "id": "正式版1.17",
        "type": "release"
    },
    {
        "id": "测试版1.17",
        "type": "snapshot"
    }
]

id是Minecraft的游戏版本，它可以被直接用于install_minecraft_version和其他需要用到游戏版本的命令。

type是这个版本的类型. 常见的值有release（正式版）, snapshot（测试版）, beta（beta版）, alpha（alpha版）.

像官方启动器一样，有时，您会想让玩家时刻玩到最新的版本，您可以使用get_latest_version()来做到这一点

最新的正式版 = minecraft_launcher_lib.utils.get_latest_version()["release"]
最新的测试版 = minecraft_launcher_lib.utils.get_latest_version()["snapshot"]

启动Minecraft

既然你已经知道了如何安装Minecraft，那么现在就可以启动Minecraft了。首先，我们需要一个包含着所有启动设置的dict。一般来讲这个dict是这么写的：

{
    "username": "游戏名",
    "uuid": "UUID",
    "token": "令牌"
}

什么是UUID和游戏名？任何一个游戏账号都有UUID和游戏名，这些东西是别人可以查到的，但登录不可能只需要别人能查到的东西。

令牌是登录时的钥匙。 用户每次使用微软账号登录时都会生成令牌。 Minecraft可以使用随便一个用户名加上一个根本不存在的令牌启动。这可以用于制作离线模式。 

options = minecraft_launcher_lib.utils.generate_test_options()

minecraft_launcher_lib.utils.generate_test_options()可以直接生成一个启动设置，这可以使开发变得简单。后面我们将详细说明如何进行微软登录。

请记住，发布能让未购买Minecraft的用户玩Minecraft的启动器是违法的！（PS：译者留，这个违法不管中国，PCL，HMCL......哪个没有离线模式）

minecraft_command = minecraft_launcher_lib.command.get_minecraft_command("1.17", minecraft_directory, options)

minecraft_command是一段命令，你可以直接使用os.system()调用它，但是不推荐。
